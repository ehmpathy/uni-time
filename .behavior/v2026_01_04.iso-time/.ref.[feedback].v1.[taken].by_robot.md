## pending decisions from vision

### decision 1: aliases

**recommendation:** drop all aliases

**rationale:**
- one canonical name per type reduces confusion
- autocomplete shows one option
- no drift between codebases

**proposed resolution:** drop aliases; use only:
- `IsoTimeStamp` (not `IsoTimestamp`, not `IsoStampTime`)
- `IsoDateStamp` (not `IsoDatestamp`, not `IsoStampDate`)

---

### decision 2: `IsoWeekdayFloat`

**recommendation:** add `IsoWeekdayFloat`

**rationale:**
- weekday patterns are common (e.g., standup on monday)
- distinct from `IsoDayFloat` (day of month)
- format: `'1'` to `'7'` (ISO 8601 weekday)

**proposed type:**
| type | format | example | semantics |
|------|--------|---------|-----------|
| `IsoWeekdayFloat` | `d` | `'1'` | day of week (floats weekly, 1=monday) |

---

### decision 3: drop `IsoYearFloat`

**recommendation:** drop it

**rationale:**
- years don't cycle in a meaningful way
- no clear use case
- fiscal year alignment is better modeled as config, not a float

**proposed resolution:** remove from the design

## final proposed types

### stamps (absolute instants)

| type | format | example |
|------|--------|---------|
| `IsoTimeStamp` | `yyyy-MM-ddTHH:mm:ssZ` | `'2024-01-15T14:30:00Z'` |
| `IsoDateStamp` | `yyyy-MM-dd` | `'2024-01-15'` |
| `IsoMonthStamp` | `yyyy-MM` | `'2024-01'` |
| `IsoYearStamp` | `yyyy` | `'2024'` |

### floats (detached patterns)

| type | format | example |
|------|--------|---------|
| `IsoTimeFloat` | `HH:mm:ss` | `'14:30:00'` |
| `IsoHourFloat` | `HH` | `'14'` |
| `IsoMinuteFloat` | `mm` | `'30'` |
| `IsoMonthFloat` | `MM` | `'01'` |
| `IsoDayFloat` | `dd` | `'15'` |
| `IsoWeekdayFloat` | `d` | `'1'` |

### ranges

| type | shape |
|------|-------|
| `IsoTimeStampRange` | `{ since: IsoTimeStamp; until: IsoTimeStamp }` |
| `IsoDateStampRange` | `{ since: IsoDateStamp; until: IsoDateStamp }` |
